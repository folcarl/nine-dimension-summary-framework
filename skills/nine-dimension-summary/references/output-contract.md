# Output Contract

Default output is Markdown.

The summary should read like a clear essay, not a form. Section names may be adapted, but the required functions must remain.

## Full Mode Sections

Use these sections unless a more natural structure serves the source better:

1. **Core Content / Core Argument**
   - Restore the source before evaluating it.
   - Identify people, events, claims, concepts, evidence, and turning points.

2. **The Real Problem It Handles**
   - State the surface topic and the deeper problem.
   - The deeper problem must not be a synonym of the surface topic.

3. **Form And Viewing Logic**
   - Identify the form: interview, lecture, report, commentary, documentary, article, etc.
   - Explain how form shapes credibility, bias, omissions, and audience response.
   - Identify the likely audience and viewing/reading risk.

4. **Evidence Chain And Reasoning**
   - Explain how the source moves from evidence to conclusion.
   - Identify key variables and transmission mechanisms where relevant.

5. **Critique And Limits**
   - Include concrete points under at least three relevant dimensions: logic errors, evidence gaps, alternative positions, framework risks.
   - Full mode should include at least three specific alternative-position objections unless not applicable.

6. **External Verification Boundary**
   - Separate source claims from unverified claims.
   - Mark transcript errors, missing charts, unclear data口径, and claims requiring external checking.

7. **Implications For Judgment And Action**
   - Explain what the material helps the reader understand or decide.
   - For economic or investment material, provide judgment variables and boundaries, not trading instructions.

8. **Follow-Up Observation List**
   - Use for evolving analytical topics.
   - List specific data, events, documents, prices, policy signals, or facts to watch.

9. **Overall Evaluation And Scores**
   - Give a one-sentence overall judgment.
   - Provide a 10-point scored table with concise reasons.
   - Full mode uses at least 7 dimensions.

## Lightweight Mode Sections

Use for short, low-density material:

1. **Core Content**
2. **Real Problem**
3. **Form / Viewing Risk**
4. **Main Limitation**
5. **Overall Evaluation And Scores**

Lightweight mode uses 3-5 scoring dimensions.

## Language

Match the user's requested output language. If none is specified, match the source language. For mixed-language inputs, choose the language that best serves the user's stated purpose.
